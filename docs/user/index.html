Quick Start

Get started with Pulse in just a few minutes:

  1. Login - Use your admin credentials to access the dashboard
  2. Add a Server - Click "Add Server" and enter your server details
  3. Configure Credentials - Add SSH credentials for your server
  4. Start Monitoring - View real-time metrics and container status

Requirements

Servers you want to monitor should have:

  • SSH access (port 22 by default)
  • A user account with sudo access (optional, for some metrics)
  • Docker installed (optional, for container monitoring)

Adding Servers

To add a new server to monitoring:

  1. Navigate to the Servers page
  2. Click the + button in the top right
  3. Fill in the server details:
    • Name - A friendly name for the server
    • Hostname/IP - The server's address
    • Port - SSH port (default: 22)
    • Description - Optional notes
    • Tags - Comma-separated tags for organization
    • Folder - Group servers by folder
    • Auth Method - Password or SSH Key
  4. Click Add to save
Tip: Use tags like "production", "development", or "docker" to organize your servers and quickly filter them later.

Status & Metrics

Each server shows a real-time status indicator:

  • Online - Server is connected and responding
  • Offline - Server is not reachable
  • Warning - Server has high resource usage

Available Metrics

  • CPU Usage - Current processor utilization
  • Memory Usage - RAM consumption percentage
  • Disk Usage - Primary disk space utilization
  • Uptime - Time since last boot
  • Load Average - System load over 1, 5, 15 minutes

Filtering & Tags

Use the filter bar to quickly find servers:

  • Search Box - Filter by name, hostname, or description
  • Status Chips - Click to show only Online, Offline, or Warning servers
  • Tag Chips - Click a tag to filter servers with that tag
  • Folder View - Toggle the folder icon to group servers by folder

SSH Keys

Pulse supports SSH key authentication for secure server access:

Generating a New Key

  1. Go to Settings
  2. Find the SSH Keys section
  3. Click Generate Key
  4. Enter a name for the key
  5. Choose the key type (RSA 4096 or Ed25519)
  6. Click Generate
Important: The private key is only shown once after generation. Copy it immediately and store it securely.

Adding Keys to Servers

After generating a key, add the public key to your server's ~/.ssh/authorized_keys file:

ssh-copy-id -i ~/.ssh/id_pulse user@server

Or manually append the public key to the file.

Credentials

Credentials are stored encrypted and associated with servers:

Credential Types

  • SSH Password - Username and password authentication
  • SSH Key - Key-based authentication (more secure)

Adding Credentials

  1. Open a server's detail page
  2. Navigate to the Credentials tab
  3. Click Add Credential
  4. Choose the type and enter the details
  5. Click Save

Authentication

Pulse supports different authentication methods per server:

  • Password - Traditional SSH password authentication
  • SSH Key - Public/private key pair authentication
  • None - For servers that don't require SSH (display only)

Alerts

Configure alerts to be notified when servers need attention:

Creating Alert Rules

  1. Navigate to Alerts
  2. Click Create Rule
  3. Configure the rule:
    • Metric - CPU, Memory, or Disk
    • Operator - Greater than, Less than, Equals
    • Threshold - The value to trigger the alert
    • Severity - Info, Warning, or Critical
  4. Click Save

Notification Channels

Alerts can be sent through various channels:

  • Email - SMTP email notifications
  • Webhook - HTTP POST to custom endpoints
  • Slack - Slack workspace integration

Docker Containers

Monitor Docker containers running on your servers:

Container Status

  • Running - Container is active
  • Stopped - Container is not running
  • Healthy - Container passed health checks
  • Unhealthy - Container failed health checks

Container Metrics

  • CPU usage per container
  • Memory consumption
  • Network I/O
  • Block I/O

Metrics History

View historical metrics for trend analysis:

  • Time Range - Select 1 hour, 24 hours, 7 days, or 30 days
  • Charts - Visual graphs for each metric type
  • Export - Download metrics data for analysis
